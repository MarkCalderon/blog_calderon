What is SASS/SCSS?

Also known as Syntactically Awesome Style Sheets. SASS is the first installment which supports CSS, while SCSS (Sassy CSS) is the newest installment which supports and uses CSS3. Both are a stylesheet language orchestrated by Hampton Catlin and developed by Natalie Weizenbaum.

It is a preprocessor or a program that converts a specific set of rules to match the requirements of another program. In this case, it is a CSS preprocessor similar to BABEL, but specifically for CSS.

Previously, what made it popular was that you could break down style sheets into different components or .scss files making it easily to manage as the complexity and project scale increases. The file is processed, compiled and outputs as a CSS. The CSS of today does not require it, and it can do that natively by using the @import function.

https://sass-lang.com/

Why should i consider using it?

Its up to your personal preference, but i personally, find that it is a worth it to be a "must have" extension to your skillset. As you move forward as a front-end developer or web designer. Though as CSS support across browsers improves over time, the use of SASS may significantly be reduced. But that is normal in any language. Every build cycle, will change it either for the better or for the worst. And that is why understanding the basics is important.

Take it as a enhancement or extension of CSS. It can help you with the right configuration, or it can be detrimental to your project.

Use cases:

  • Improves readability.
  • Ability to use mixins or functions to do logical decisions
  • Ease of recycling and adapting to different coding scenarios

Examples:

Flex Box

            
                

                @mixin flex($jc: space-between, $ai: flex-start, $fd: row, $ff: row wrap) {
